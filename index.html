Running a self-hosted Plex Media Server on dedicated home lab hardware to serve a personal media library locally and remotely — without relying on third-party streaming services or cloud storage.

Setup

Plex runs on a Linux-based home server with local storage attached for the media library. Cloudflare Tunnels — Secure Remote Access

All externally accessible home lab services — Plex and Home Assistant — are exposed to the internet exclusively via Cloudflare Tunnels. No ports are open on the home router. All inbound traffic routes through Cloudflare's global edge network, which handles TLS termination, DDoS protection, and access enforcement before any request reaches the home network.

How It Works

The cloudflared daemon runs on the home server and maintains persistent outbound connections to Cloudflare's edge. Incoming requests to the configured hostnames are routed through these tunnels back to the relevant local service. Because the connection is outbound-initiated, there is no need to expose any port or configure dynamic DNS — Cloudflare handles routing entirely.

Zero Trust Access

Services are further protected by Cloudflare Zero Trust access policies, which enforce identity verification before a request can reach the tunnel. This means that even if a hostname is discovered, unauthenticated access is blocked at Cloudflare's edge — not just at the application level. The result is a secure, zero-port-exposure remote access architecture that applies the same Zero Trust principles used in enterprise environments to a home lab context.
